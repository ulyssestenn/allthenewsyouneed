## Note

Selection window: 2026-07-22T13:37 UTC through 2026-07-23T13:37 UTC. All eight WSJ Dow Jones RSS feeds, all ten NYT section RSS feeds, and all eight NBC section RSS feeds listed in the source instructions returned HTTP 200 and were used to build each outlet's candidate pool; items were filtered on `pubDate`, not on feed-retrieval time. AP does not publish a public RSS feed, so its candidates were initially sourced via Google News's `site:apnews.com when:1d` search feed (also HTTP 200), but that feed's `news.google.com/rss/articles/...` links resolve into a client-side-rendered Google interstitial rather than redirecting to apnews.com, so they could not be used as canonical URLs (confirmed via curl with a persistent cookie jar and header inspection). Instead, AP's own homepage (apnews.com) was fetched directly and its embedded `schema.org` `ItemList` structured data — a ranked list of AP's own top homepage placements — was used to identify candidate articles; each candidate's canonical URL, headline, and exact `article:published_time` metadata were then verified by fetching the article page directly from apnews.com. WSJ article pages return a bot-protection placeholder to automated requests without a browser session, so WSJ links could not be live-verified by fetching the page directly — they are the exact, unmodified `<link>` values published in WSJ's own official RSS feeds, with WSJ's feed-tracking query parameters (`?mod=...`) stripped for a clean canonical URL. One WSJ item (the DOJ merger-review story) carried a tweet-style caption rather than a formal headline in the feed's `<title>` field; the headline used here was reconstructed from the article's URL slug, consistent with WSJ's standard headline for that story.
